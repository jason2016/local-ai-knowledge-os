# CLAUDE.md — Local AI Knowledge OS

Guidance for Claude (and any AI agent) working in this repository. Read this first before creating or editing files.

## 1. Project purpose

**Local AI Knowledge OS** is the public companion repository for the future KDP book *Build Your Local AI Knowledge OS* and for the **Semantic OS Method**.

It exists to give readers a practical, copy-ready starting point: methodology, starter-kit structure, templates, and safe examples for building a local-first, Markdown-based, AI-readable knowledge system. It is the public entry point to the method — not the book itself.

## 2. Public safety rules

**This repository is public. Everything committed here is visible to the world.**

- Treat every file as if it will be read by anyone, forever.
- When in doubt about whether something is safe to publish, leave it out and ask.
- Never paste content from private notes, client work, or AI chat logs without first stripping anything identifying or sensitive.
- Keep all content generic, safe, practical, and reusable.

## 3. Allowed content

- Methodology and explanations of the Semantic OS Method (public-safe level).
- Starter-kit folder structures and their READMEs.
- Templates (decision records, incident records, project maps, context packs, action feedback, etc.).
- Safe, fictional examples that use invented projects and data.
- Companion resources for the book (outlines, overviews, checklists).
- General documentation written for a public GitHub audience.

## 4. Forbidden content

Never include:

- Private customer or client data of any kind.
- API keys, tokens, passwords, or any credentials.
- Private business records, internal financials, or confidential plans.
- Private or real AI conversation logs.
- Full or substantial KDP manuscript text.
- Anything that identifies a real person or organization without permission.

If asked to add any of the above, refuse and explain the public-repo boundary.

## 5. Directory rules

- Keep the repository **simple and practical**. Do not create unnecessary top-level directories.
- Use the existing structure:
  - `docs/` — methodology and overviews
  - `starter-kit/` — copy-ready folder structure (`10_AI_OS/...`)
  - `templates/` — reusable Markdown templates
  - `examples/` — safe, fictional examples
  - `book/` — book companion materials (e.g. `book/outline/`)
- Place new files in the most fitting existing folder before proposing a new one.
- If a new directory seems needed, ask first and justify it.

## 6. Writing style

- **Prefer Markdown** for all content.
- **Prefer dated filenames** for major documents: `YYYY-MM-DD-title.md` (e.g. `2026-06-07-method-overview.md`). Stable reference files (READMEs, templates) need not be dated.
- Write clearly and practically for a general public GitHub audience.
- Use relative Markdown links (`[text](path)`) so references work in any editor and for any AI.
- Keep templates copy-ready: clear sections, generic placeholders, no private examples.
- Be concise. Favor lists, tables, and short sections over walls of text.

## 7. Git rules

- **Do not run `git commit` unless explicitly asked.**
- Do not push, force-push, or alter history unless explicitly asked.
- After making changes, show a concise summary of what was created or updated and stop.
- Let the user review before any commit.

## 8. KDP book boundary

- The **full book manuscript must remain private.** It does not belong in this repository.
- This repo contains only public companion material: methodology overviews, structure, templates, safe examples, and outlines.
- Outlines and summaries are fine; chapter-length manuscript text is not.
- Always frame the repo as a **companion resource**, not the book itself.

## 9. Semantic OS Method boundary

- The **Semantic OS Method** is the internal methodology behind this project.
- Share the public-safe, practical layer: principles, structure, and reusable templates.
- Do not publish proprietary internal details, private playbooks, or client-specific implementations.
- When explaining the method, keep it generic and reusable — the depth lives in the book, the practical starter lives here.
