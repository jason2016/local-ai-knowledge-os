# Starter Kit — Local AI Knowledge OS

This is a minimal, ready-to-copy structure for your own **Local AI Knowledge OS**. Copy the `10_AI_OS/` folder into your notes vault (Obsidian, a plain folder, a Git repo — anything that holds Markdown) and start using it today.

## What's inside

```
starter-kit/
└── 10_AI_OS/
    ├── 00_INBOX/        # Raw capture — everything lands here first
    ├── 01_Distill/      # Process raw input into meaning
    ├── 02_Knowledge/    # Durable notes you want to keep and reuse
    ├── 03_Index/        # The map / table of contents of your OS
    ├── 06_Projects/     # Active projects, each with a Project Map
    └── 09_AI_System/    # Rules and context that tell AI how to use the OS
```

> The numbers (00, 01, 02, 03, 06, 09) keep folders in a stable, predictable order and leave room to add more later (e.g. `04_`, `05_`, `07_`, `08_`). They are part of the convention — keep them.

## How to use it

1. **Copy** `10_AI_OS/` into your vault or working folder.
2. **Capture** anything new into `00_INBOX/` without worrying about organization.
3. **Distill** inbox items in `01_Distill/` — extract what matters.
4. **Promote** durable insights into `02_Knowledge/`.
5. **Map** your work in `06_Projects/` using the [Project Map template](../templates/Project_Map_Template.md).
6. **Point your AI** at `09_AI_System/` plus a Context Pack, and let it work.
7. **Record outcomes** with the [Action Feedback template](../templates/Action_Feedback_Template.md).

## The flow

```
Capture → Distill → Keep → Index → Project work → Feed AI → Feedback → (repeat)
```

## Principles

- **Local-first.** Your files, your machine, your control.
- **Plain Markdown.** Readable by you and by any AI model.
- **Role over topic.** Each folder defines what a file *is*, not just what it's about.
- **Small and copyable.** Start minimal; grow only when you feel the need.
