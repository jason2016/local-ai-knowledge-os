# Visual Direction — Local AI Knowledge OS

This folder holds the lightweight visual layer for the project: diagrams and the guidance that keeps them consistent. The goal is a **calm technical field guide**, not an AI startup. Think *operator manual*, not *AI revolution*.

All visuals here are public-safe: no private data, no real AI conversations, no fake analytics.

## Visual positioning

> Plain-text knowledge infrastructure for serious AI users.

The visuals should reassure a careful reader that this is a stable, owned, understandable system — files you control, not a hosted product chasing trends.

## Visual identity

- **Practical** — every diagram earns its place by explaining something.
- **Calm** — quiet, uncluttered, low contrast where possible.
- **Clear** — one idea per diagram; readable at a glance.
- **Trustworthy** — honest, no overpromising.
- **Technical but understandable** — precise without jargon overload.
- **Lightweight & local-first** — simple shapes, plain text, no heavy infrastructure.

## Color palette

Use sparingly. Most diagrams are fine in plain black-on-paper; color is an accent, not decoration.

| Role | Name | Hex | Use for |
|------|------|-----|---------|
| Background | Paper | `#F7F4EE` | Page / canvas background |
| Primary text | Graphite | `#202124` | Body text, labels, lines |
| Accent (system) | Ink blue | `#1F4E79` | The OS / structure nodes |
| Accent (local) | Local green | `#4F7D5A` | Local-first / knowledge nodes |
| Accent (action) | Soft amber | `#C99735` | Actions, feedback, highlights |
| Lines | Line gray | `#C9C3B8` | Borders, connectors, dividers |

Keep to one or two accents per diagram. When in doubt, use fewer colors.

## Typography direction

- **Headings & UI:** a clean humanist sans-serif (e.g. system UI sans).
- **Body / explanation:** the same sans-serif at a comfortable reading size.
- **Code, paths, file names:** a monospace face (e.g. system mono).
- Favor generous spacing and short lines. Calm beats dense.

## Diagram style

- Prefer **Mermaid** in Markdown — diffable, editable, version-controlled, and rendered by GitHub.
- One concept per diagram. If it needs a legend to be understood, simplify it.
- Left-to-right flows for processes; simple trees for structure.
- Plain rectangles and arrows. Avoid clutter, gradients, and effects.
- Label nodes in plain language a first-time reader understands.

## What to avoid

- Generic AI robot imagery, glowing brains, neural-network meshes.
- Cyberpunk or dark "hacker" aesthetics.
- "Second brain" clichés.
- Enterprise SaaS dashboards, fake analytics, growth charts, or ROI visuals.
- Overpromising automation imagery.
- Heavy local-LLM infrastructure diagrams.
- Too many tool logos.
- Any private customer or project data, or screenshots of real AI conversations.
