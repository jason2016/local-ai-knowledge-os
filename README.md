# Local AI Knowledge OS

A local-first AI knowledge operating system for turning Markdown notes, project records, AI conversations, decisions, incidents, and action feedback into reusable context for Claude, Codex, and other AI agents.

This repository is the companion project for the **Local AI Knowledge OS** method, also known internally as the **Semantic OS Method**.

The goal is simple:

> Turn scattered project experience into a structured, AI-readable knowledge system that can support real work, long-term memory, project execution, and continuous learning.

## Why this project exists

AI tools such as Claude, Codex, ChatGPT, and local LLMs are powerful, but they often lose context across projects.

Most people already have valuable knowledge in:

- Markdown notes
- Obsidian vaults
- AI conversations
- Project documents
- Deployment records
- Bug reports
- Decisions
- Customer cases
- Daily work logs

But this knowledge is usually not structured in a way that AI agents can reliably read, reuse, and act on.

Local AI Knowledge OS provides a lightweight structure for organizing this knowledge.

## Folder meaning

### docs/

Methodology documents, explanations, diagrams, and technical notes.

### starter-kit/

A minimal folder structure that users can copy to start their own Local AI Knowledge OS.

### templates/

Reusable Markdown templates for project records, decisions, incidents, action feedback, and context packs.

### examples/

Safe public examples showing how the system can be used without exposing private business data.

### book/

Public companion materials for the KDP book.

The full manuscript is not stored in this public repository.

## Important privacy note

This repository is designed for public examples and reusable templates only.

Do not commit private project data, customer data, business secrets, API keys, personal documents, or full AI conversation logs.

Private working data should stay in a local vault or private repository.
